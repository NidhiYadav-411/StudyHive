Category 1: Frontend & DOM Manipulation
Q1: What is the DOM, and how does the browser parse HTML?
Concept: DOM (Document Object Model) HTML document ka ek tree-like structure representation hota hai. Browser HTML files ko parse karke tokens banata hai, tokens se nodes banata hai, aur fir pure DOM Tree ko ready karta hai. Usi time CSS ko parse karke CSSOM tree banta hai. Dono milkar Render Tree banate hain jo final display paint karta hai.
StudyHive Context: Jab browser notes.html load karta hai, tab html ke main elements load ho jate hain, fir javascript chal kar dynamic note cards isme insert karti hai.
Interview Pro-Tip: DOM ko JS aur HTML ke beech ka path batayein. HTML parse single-threaded hota hai, isiliye large script tags top par browser loading ko block kar dete hain.
Q2: Explain the difference between DOMContentLoaded and window.onload.
Concept: DOMContentLoaded tab trigger hota hai jab HTML document parse ho jaye aur DOM tree build ho jaye (bina images aur css file ka wait kiye). window.onload tab trigger hota hai jab images, scripts, frames sab-kuch completely load ho chuke hon.
StudyHive Context: Humne `notes.js` me DOMContentLoaded event listen kiya hai taaki page ready hote hi server se notes dynamic fetch hona start ho jaye.
Interview Pro-Tip: Bataiye ki DOMContentLoaded use karne se page loading fast lagti hai (perceived speed increase hoti hai) kyunki hum resources pehle load kar dete hain.
Q3: How does StudyHive fetch and display notes dynamically? Explain the JavaScript workflow.

Interview Pro-Tip: Interviewer ko flow batayein: Request -> Route -> DB query -> JSON Response -> DOM rendering wrapper. Ye client-server communication ki deep understanding show karta hai.
